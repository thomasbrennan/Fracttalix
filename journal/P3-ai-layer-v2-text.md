## CLAIM REGISTRY

Claim ID:  D-3.1
Type:      D  [TYPE:D]
Name:      FRM System Definition
Section:   Section 2
Statement: A system S is an FRM-scope system iff: (a) S exhibits sustained or transiently-decaying oscillation; (b) the oscillation is generated by delayed negative feedback with a single dominant delay tau_gen > 0; (c) tau_gen is independently measurable (i.e. measurable by methods not derived from the FRM period prediction T = 4*tau_gen). Condition (c) is the independence requirement and is the FRM solution to the DQ problem.
Status:    LIVE
Note:      Type D — definitional claim. Not subject to falsification predicate. Gate: any system cited in P3 validation must satisfy all three conditions.

Claim ID:  D-3.2
Type:      D  [TYPE:D]
Name:      tau_gen Extraction Protocol
Section:   Section 3
Statement: tau_gen is extracted from a system S by the following procedure: Step 1 — identify the biological or physical mechanism producing delayed negative feedback. Step 2 — measure the delay of that mechanism using methods independent of the oscillation period T. Step 3 — record tau_gen with units and measurement method. Step 4 — confirm tau_gen > 0. The extracted tau_gen is the sole parameter input to the FRM prediction T = 4*tau_gen. No fitting is performed.
Status:    LIVE
Note:      Type D — protocol definition. Alpha sub-rule (HR-3.1): alpha is a system property, not a protocol parameter. Default alpha = -1 is the dimensionless normalisation for sub-critical systems (P1 scope boundary). Literature lookup is preferred source for alpha. See Section 3 prose note.

Section:   Section 4

Status:    LIVE — PHASE-READY

R2_frm: SCALAR — coefficient of determination from protocol R1-R9 step outputs. Computed as 1 - SS_res/SS_tot.
R2_best_alt: SCALAR — maximum R2 across alternative models evaluated per C-3.ALT protocol.
EVALUATION: Step 1: for each system in validation_set, apply D-3.2 to extract tau_gen. Step 2: compute derived parameters (lambda = |alpha|/(Gamma*tau_gen), omega = pi/(2*tau_gen), T_char = 4*tau_gen). Step 3: apply 9-step protocol R1-R9. Step 4: compute R2_frm. ## REGRESSION PROTOCOL (R1-R9)

Protocol ID:          R1-R9
Name:                 P3 Nine-Step Regression Protocol
IR Compliance:        IR-12 — each step has named input, deterministic operation, named output
Statistical Anchor:   IR-13 — Bevington and Robinson 2003 as primary statistical authority
Zero Free Parameter:  SATISFIED — all FRM parameters derived from tau_gen and IR rules. No fitting of FRM parameters.

Steps:
R1
Input:     Time series data, system identifier
Operation: Apply D-3.1 scope check — verify conditions (a)(b)(c)
Output:    PASS/FAIL scope verdict. FAIL exits protocol.

R2
Input:     System identifier, biological/physical substrate description
Operation: Apply D-3.2 tau_gen extraction — measure delay by substrate-appropriate independent method
Output:    tau_gen (units: time), measurement method description

R3
Input:     tau_gen, substrate literature
Operation: Determine alpha — look up bifurcation distance from literature. If unavailable, use default alpha = -1 (dimensionless sub-critical normalisation). Alpha sub-rule: alpha is a system property, not a protocol parameter.
Output:    alpha (dimensionless)

R4
Input:     tau_gen, alpha, Gamma = 1 + pi^2/4 (derived)
Operation: Compute derived parameters: lambda = |alpha|/(Gamma*tau_gen), omega = pi/(2*tau_gen), T_char = 4*tau_gen
Output:    lambda (units: 1/time), omega (units: rad/time), T_char (units: time)

R5
Input:     T_char, time series
Operation: Set observation window T_window >= 3*T_char. Extract time series segment.
Output:    Time series segment of length >= 3*T_char

R6
Input:     Time series segment, derived parameters lambda, omega
Operation: Fit FRM form f(t) = B + A*exp(-lambda*t)*cos(omega*t + phi) with B, A, phi as the only fitted parameters (3 free parameters — offset, amplitude, phase). lambda and omega are NOT fitted.
Output:    B, A, phi (fitted); residuals; R2_frm

R7
Input:     Residuals from R6
Operation: Compute sigma_S per C-3.sigma protocol (Bevington-Robinson §3.2). KS normality test.
Output:    sigma_S (substrate-specific), normality verdict

R8
Input:     Time series segment
Operation: Fit three alternative models with all parameters free: (1) B + A*exp(-lambda*t); (2) B + A*cos(omega*t + phi); (3) B + c1*t + c2*t^2. Compute R2 for each.
Output:    R2_best_alt = max of three alternative R2 values

R9
Input:     R2_frm, R2_best_alt, threshold 0.85
Operation: Evaluate C-3.REG predicate: (R2_frm >= 0.85) AND (R2_frm > R2_best_alt). Evaluate C-3.ALT predicate: R2_frm > R2_best_alt.
Output:    PASS or FAIL verdict with supporting values
